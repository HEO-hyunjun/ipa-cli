# docs

Historical planning documents. These capture how a feature was designed, not
how it currently works — for live reference use `README.md` (user-facing),
`ARCHITECTURE.md` (layout), `CLAUDE.md` (design rules), and `ipa convention`.

## What lives here

- `superpowers/plans/` — the implementation plan(s) for the behavioral
  benchmark effort.
- `superpowers/specs/` — the design spec(s) behind that effort; `bench/README.md`
  points at the bench design spec for background.

Kept for provenance. If a claim here conflicts with the code, the code wins.
